# Колони-сим — интерактивная модель (v2)

> Дизайн перехода от макро-абстракции (Фазы 0–6: «локализуй узел + колонисты») к тактильному
> логистическому колони-симу: поштучные заказы слайдерами, табы Земля/Марс, полный физический
> слой стоков/потоков. Тезис не меняется — он становится **ощутимым** через дефициты и массу.
> Истина по числам/механике — `mechanics.md`, `graph.md`; решения — D-041…D-044. Статус: дизайн.

---

## 1. Зачем (диагноз)

Макро-петля честна, но не тактильна: игрок крутит абстракции, не «чувствует» колонию. v2 даёт
**физический логистический слой** — заказываешь поштучно (тонны/штуки) слайдерами, видишь живые
балансы («на N людей нужно X еды — у тебя Y», «теплицам нужно больше энергии, чем генерация»),
строишь инфраструктуру. Стена MES (чипы/фарма/катализаторы нелокализуемы) проявляется не цифрой
автономии, а **вечным завозом критичных грузов** и дефицитами при его обрыве (запас хода D-025).

## 2. Физический слой — ресурсы, стоки, потоки (D-042)

Над денежным ledger (D-027 цел — деньги остаются валютой заказа) надстраивается **по-ресурсный
учёт**. Каждое окно: `сток' = сток + производство + прибытие − потребление`; дефицит → каскад.

| Ресурс | Тип | Источник | Потребитель | Дефицит → |
|--------|-----|----------|-------------|-----------|
| **Энергия** | поток (ген↔спрос) | электростанции | ЖО, фермы, заводы, хабитаты | браунаут: фермы/ЖО/заводы режут выпуск → каскад |
| **Еда** | сток (т) | фермы (энергия+вода+семена) | население | голод → смертность |
| **Вода** | сток (т) | ISRU-лёд + рециклинг (ECLSS η<1) + завоз | люди + фермы | смертность |
| **Газы** (O₂/N₂) | сток | MOXIE/ISRU + рециклинг (η<1) + завоз | дыхание + утечки | смертность |
| **Материалы/спейрс** | сток | заводы локализации + завоз | износ/ремонт узлов | поломки (каскад §5.6) |
| **Деньги (M)** | бюджет | субсидия Земли (инфляция D-031) | все заказы | отмена проекта (§7.4) |

**Каскад дефицита** (делает тезис ощутимым): энергия < спрос → браунаут → фермы/ЖО/заводы режут
выпуск → еда/вода/воздух падают → смертность и гашение узлов → ещё меньше выпуска. Это §5.6/§6.5
в физической форме. КПД рециклинга η<1 (ECLSS, references §2) → вечная подпитка по воде/газам.

## 3. Петля хода (синодическое окно)

1. **Планирование на Земле (заказ).** Табы (§4), слайдеры количеств. Живьём: суммарная масса vs
   **пропускная способность запуска** (§5), суммарная стоимость vs бюджет `M`, балансы-подсказки
   (хватит ли еды/энергии/воды на прогноз населения и структур).
2. **Планирование на Марсе (стройка).** Очередь строительства структур/заводов (§6) из прибывших
   материалов + локального производства; пререквизиты проверяются (АЭС→площадка отходов и т.д.).
3. **Коммит → транзит (лаг).** Заказы уходят, **прибывают в следующее окно** (тирания Циолковского,
   §4.4). Стройка идёт окно-два.
4. **Разрешение окна.** Производство/потребление/стоки пересчитываются; дефициты каскадят; износ
   ест спейрс; события (§12); население меняется.
5. Повтор. Тезис эмёрджентен: чёрные узлы построить нельзя → их грузы завозятся вечно → масса+деньги
   стена; обрыв завоза → дефициты за ~0.5 окна (запас хода пригвождён, D-025).

## 4. Земля — табы заказа (D-041)

Каждый таб — карточки с иконкой, слайдером (штуки/тонны), удельным весом и ценой, живым итогом.

1. **Пуск / логистика (§5):** посадочные площадки (классические; после RnD — под орбитальную
   заправку), ракеты (классические многоступенчатые → многоразовые), топливо.
2. **Люди:** специалисты по специальностям (инженеры ЖО, агрономы, медики, металлурги, химики,
   операторы АЭС, учителя, исследователи) + колонисты общего профиля. Вес/чел + вечный шлейф
   потребления (D-030) + конвейер обучения (школы, D-028).
3. **Жизнеобеспечение:** еда, вода, газы (O₂/N₂) — расходники-завоз, пока не локализовано.
4. **Материалы:** сталь, конструкц. металлы, стекло/керамика, полимеры, **спейрс** (ремонт).
5. **Оборудование:** станки, модули заводов, теплицы, электростанции (солнечные/ядерные/РИТЭГ/
   турбинные), батареи. Электростанции — вес + (ядерные) требование площадки отходов.

## 5. Логистика запуска (D-043 — явная форма D-038, лестница D-039)

- **Площадки** — капитал: строятся, простаивают 25/26 мес между окнами (бёрст, D-038). Каждая →
  1 пуск/день; окно = 5 пусковых дней → **5 пусков/площадку/окно**. Старт: 5 классических → 25 пусков.
- **Ракеты** — груз на пуск (payload, кг). Классические многоступенчатые = одноразовые (Gen0, D-039).
  После RnD — многоразовые + орбитальная заправка (Gen1): дешевле кг, но стена остаётся (§4.6).
- **Топливо** — расходник на пуск.
- **Пропускная способность/окно = площадки × пуски × payload.** Заказ нельзя превысить — либо строй
  больше площадок (капитал + простой), либо режь завоз (дефицит). Это «сжатие триллиона» физически.

## 6. Марс — стройка структур (D-044)

Структуры — узлы с: capex (масса/ISRU), апкип (спейрс+энергия), энергобаланс (ген/спрос), выпуск
(ресурс или локализованный узел графа), **пререквизиты**, штат (специалисты, D-034).

- **Электростанции:** солнечные (дёшево, переменно), ядерные (+ **площадка отходов**), РИТЭГ, турбинные.
- **Фермы/теплицы:** энергия + вода + семена → еда. «Теплицам нужно больше энергии, чем генерация» — гейт.
- **ЖО-модули:** рециклинг воды/воздуха (η<1, references §2).
- **Медблоки:** требуют **фарма (чёрный!)** → вечный завоз. **Роддома:** энергия + медицина → рождения (D-030).
- **RnD-лаборатории:** открывают лестницу запуска (D-039) / мегапроекты (D-033).
- **Школы:** конвейер обучения специалистов (D-028).
- **Заводы локализации:** строят узлы графа (`graph.md`) — но чёрные построить нельзя (MES, §1.3).
- **Посадочные площадки, хранилища отходов** (ядерные/РИТЭГ/турбинные).

## 7. Реконсиляция с тезисом и решениями

- **Тезис цел:** чёрные узлы (чипы/фарма/катализаторы/прецизионка) нельзя построить ни заводом, ни
  RnD → медблоки/заводы/фермы вечно тянут их завозом → масса+деньги стена; обрыв → дефицит за ~0.5 окна.
- **D-027** (единый денежный ledger) цел — деньги остаются валютой заказа; физические стоки — поверх.
- **D-025** автономия (по массе) vs запас хода — теперь запас хода = буквально «сколько окон стоки
  продержатся при обрыве завоза», считается из стоков напрямую (честнее прежнего прокси).
- **D-038/D-039** пусковая мощность — теперь явные площадки/ракеты/топливо/лестница.
- **D-030/D-034/D-028** население/специалисты/обучение — явные заказы и структуры.
- Движок Фаз 0–6 (узлы, MES, износ, события, инфляция) — **субстрат**; над ним добавляется
  ресурсный слой и заменяется player-facing петля. Не выбрасываем — наслаиваем.

## 8. Открытые вопросы (уточнить до билда)

- Список ресурсов финален (энергия/еда/вода/газы/спейрс) или дробим (O₂/N₂ отдельно, разные материалы)?
- Гранулярность специальностей: сколько специальностей и их роль в выпуске структур?
- Стройка на Марсе: из чего строим структуры — завезённые модули (масса) или местные материалы (ISRU) или микс?
- Браунаут: жёсткое отключение по приоритету (ЖО→еда→заводы) или пропорциональное снижение выпуска?
- Темп: окно по-прежнему ~26 мес, или для тактильности нужны под-ходы внутри окна?

## 9. Набросок фаз билда (после фиксации модели)

1. **Ресурсный слой движка:** стоки/потоки/балансы + каскад дефицита; золотые тесты на сохранение тезиса.
2. **Логистика запуска:** площадки/ракеты/топливо/пропускная способность (замена абстрактного `K`).
3. **Земля-таб:** табовый ордер-UI со слайдерами, живыми балансами и лимитом массы.
4. **Марс-таб:** очередь стройки структур с пререквизитами и энергобалансом.
5. **Структуры-контент:** электростанции/фермы/ЖО/медблоки/школы/лаборатории/хранилища.
6. **Дебриф v2:** запас хода из реальных стоков; кривые дефицитов.
