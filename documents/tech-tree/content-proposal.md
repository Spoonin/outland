# Содержание дерева — ресурсы, структуры, техи, эффекты

> Конкретное наполнение: что добавить в `resources.csv`, `structures.csv`, `techs.csv` и какие
> новые эффекты нужны движку. Числа — TOY (порядки; калибровка плейтестами, как resources.csv).
> Привязка к `graph.md` (узлы v1) указана в скобках.

## Главный дизайн-принцип: гейт, а не бонус

Дерево технологий — это в основном **`unlockStructure`-гейт** над сильно обогащённым
`structures.csv`. Реализм живёт в **BOM-цепочках структур** (как в graph.md), а не в размножении
скалярных `TechEffect`. Технология = «теперь можно строить объект X», а X имеет реалистичные входы.
Лишь горстка эффектов (труд/η/рождения/радиация) требует настоящих движковых хуков.

Почему так: CSV — источник правды (D-058); цепочка `regolith → metals → steel → machine → spares`
даёт всю глубину сама, без нового кода на каждый шаг. Движок трогаем минимально.

## Второй дизайн-принцип: цикл, а не one-shot (анализ 2026-07-09; истощение и кривая освоения ПРИНЯТЫ — D-087)

Системный риск базового плана: почти каждая фаза устроена как «тех → структура → брешь закрыта
НАВСЕГДА». Это рождает шаблонные ходы — однажды найденный порядок построек становится решением на
все партии, а закрытые бреши не возвращаются. В реальности ни одна индустриальная проблема не
решается один раз: **возобновляющиеся жизненные циклы** (истощение, деградация, убыль кадров,
освоение процесса) превращают разовые анлоки в постоянное давление. Прецеденты в самой игре:
демография D-083 (волна старения основателей) и порча D-085 дали «второй темп» почти без роста
интерфейсной сложности. Вращающееся горлышко (`path-to-100k.md` §3) работает, только если старые
горлышки не решены навсегда.

Три механизма одного семейства «плоское число → медленная функция накопления», на трёх разных
горизонтах (ресурсный / переходный / поколенческий). Интерфейсно почти бесплатны (хроника D-061
уже умеет причинность); брать все пять идей ниже не обязательно — этих трёх достаточно:

1. **Истощение месторождений (P1).** `ice_mine`/`excavator` не плоские источники навсегда:
   накопленная добыча по типу → растущая цена энергии/труда за кг (или дискретно: «жила
   исчерпана, следующая +40% энергии»). Одно число на тип; добывающий слой перестаёт быть
   «поставил N штук и забыл». Реальность: ближняя жила вырабатывается, следующая дальше и беднее
   (`real-industrial-stages.md` §1.2 «привязка к месту залегания» — переведено в игровую форму).
2. **Кривая освоения производства (P2–P4).** Новый процесс первые окна работает на 30–50%
   выхода, `yield` растёт с накопленным выпуском типа (learning curve — реальный индустриальный
   закон). Убивает шаблон «копи деньги → купи тех в момент X»: локализоваться надо заранее и с
   запасом; переходный период (импорт уже срезан, местное ещё сырое) — самое опасное окно.
3. **Специалисты — атрибут колониста, не пул (P6).** См. секцию P6: квалификация умирает вместе
   с человеком (D-083 делает это почти бесплатным) — реальный кадровый цикл, образование нельзя
   выключить никогда.

Дешёвый флейвор без нового кода: **перхлораты** — постоянная строка BOM фермы (вода/энергия на
промывку грунта), не one-shot тех «биоремедиация» (см. «что не механизировать»).

Больший скоуп (отдельная дизайн-сессия): **вариативность площадки** — 3–4 сид-параметра
(глубина льда, перхлораты, широта/солнце, качество руды) сдвигают относительные цены и ломают
заученный опенер; диегетично как «отчёт геологоразведки» (D-036). Анти-шаблонный ход №1 —
вопрос зафиксирован в `implementation-roadmap.md`.

## Новые ресурсы (фазированно, курировано)

13 текущих → +6 к концу. Не все сразу: по фазам, чтобы не перегружать дашборд. `earthPerKg` для
местных промежуточных — высокий/запретный (их не завозят, только производят локально; см. «только
локально» ниже).

| id | появл. | earthPerKg | perCapita | recycle | что это (graph.md) | роль |
|----|--------|-----------:|----------:|--------:|--------------------|------|
| `regolith` | P1 | — (только локально) | 0 | 0 | `regolith` | феедсток металлов/стекла/стройматериалов |
| `hydrogen` | P2 | — (только локально) | 0 | 0 | `hydrogen` (электролиз) | восстановитель + сырьё химии |
| `co2` | P1 | — (только локально) | 0 | 0 | `co2` (атмосфера) | источник углерода |
| `composite` | P3 | дорого (импорт бессмыслен) | 0 | 0 | `ceramics`/базальт | местный объёмный стройматериал (реголит-печать) |
| `components` | P3/P4 | дорого (можно завезти узлы, D-057) | 0 | 0 | (передел, `manufacturing-chain.md`) | передельный/фабрикованный продукт — структуры строятся из него, а не из сырого металла |
| `specialists` | P6 | — (не завозится дёшево) | 0 | 0 | (D-028 навыки) | квалифицированный труд, гейт хайтека |

**`components` — недостающий передел (брешь №2b, `manufacturing-chain.md`).** «Из слитка никто не
строит здание»: между материалом и структурой лежит прокат/аддитивка/фабрикация. Свёрнуто в один
ресурс `components` (передельный продукт + узлы). **Структуры строятся из `components`+`composite`,
а не из сырого `steel`/`glass`** (объём — реголит-`composite`, детали/машины — `components`). До
постройки `fab_shop` (P4) `components` можно завозить готовыми узлами (D-057) — плавный старт.

**«Только локально»** — новый флаг ресурса (`localOnly` в resources.csv): движок/стор не дают
заказать его с Земли (нет цены доставки; в `previewOrder` он не участвует). Промежуточные ISRU
(`regolith`/`hydrogen`/`co2`) физически абсурдно возить — они производятся и потребляются на Марсе.

**`metals` перестаёт быть импорт-онли.** Остаётся в каталоге (можно ещё завозить как затравку),
но теперь его ПРОИЗВОДИТ местный `mre_plant`/`iron_reducer` из `regolith` (см. P2). Это чинит
«сталь из импортных слитков» — самую абсурдную брешь.

## P1 — Первичная добыча (структуры)

Открывает поток сырья. Пока ничего конечного — входы для P2.

| id | icon | что делает | BOM (produces/consumes) | гейт |
|----|------|-----------|--------------------------|------|
| `excavator` | ⛏ | копает реголит | +regolith; −энергия | tech `isru_extraction` |
| `ice_mine` | 🧊 | добыча льда → вода | +water(первичная!); −энергия | tech `isru_extraction` |
| `co2_capture` | 🌀 | захват/сжатие атмосферы | +co2; −энергия | tech `isru_extraction` |

Эффект: вода перестаёт быть чисто завозной — `ice_mine` даёт первичную воду (сейчас
`water_recycler` только рециклит уже имеющуюся). Это отвязывает воду от Земли — первый шаг к балку.

Ограничитель по реальности: экскаватор жрёт энергию и ЗИП (абразив, D-052); лёд привязан к
энергии добычи. **Плюс истощение** (второй дизайн-принцип): накопленная добыча типа удорожает
следующий кг (энергия/труд) — добыча остаётся живым решением на всех тирах населения, а не
one-shot слоем.

## P2 — Первичная химия/металлургия (структуры)

Здесь `metals` становится местным, появляется H₂, оживает C1-химия.

| id | icon | что делает | BOM | гейт |
|----|------|-----------|-----|------|
| `electrolyzer` | ⚡💧 | электролиз воды | +hydrogen +o2; −water −энергия(много) | tech `electrolysis` |
| `mre_plant` | 🔥⚗ | MRE: металлы из реголита | +metals +o2; −regolith −энергия(много) | tech `regolith_metallurgy` |
| `iron_reducer` | 🏭 | восстановление железа H₂ | +metals; −regolith −hydrogen −энергия | tech `regolith_metallurgy` |
| `rwgs_ft` | 🧪⛓ | RWGS+Фишер-Тропш | +polymers +fuel; −co2 −hydrogen −**catalyst** | tech `c1_chemistry` |

Теперь цепочка честная: `regolith → (mre) → metals → (steel_plant) → steel`. `polymer_plant`
переводится на местный вход (`co2`+`hydrogen`+`catalyst`) вместо импортного `steel`.
**Катализатор остаётся импортным** (PGM, чёрный узел) — вход в химию, вечный пол (D-045).

Ограничитель по реальности: электролиз/MRE — главные потребители энергии (стимул к атому/термояду);
органика упирается в импортный катализатор.

## P3 — Строительство из реголита (структуры + сдвиг BOM)

Ключевой сдвиг: объекты дешевеют, объёмное жильё перестаёт требовать стали.

| id | icon | что делает | BOM | гейт |
|----|------|-----------|-----|------|
| `sinter_plant` | 🧱 | спечённый реголит/серобетон | +composite; −regolith −энергия | tech `regolith_construction` |
| `regolith_printer` | 🖨🧱 | 3D-печать конструкций | (снижает build-стоимость объектов) | tech `regolith_construction` |
| `shield_berm` | 🛡🧱 | радиационная защита | (снимает штраф радиации) | tech `regolith_construction` |

**Сдвиг стоимости постройки:** новые дешёвые версии жилья/складов используют `composite` (объём)
+ `components` (узлы/каркас) вместо сырого `steel`+`glass`. Например, `habitat_regolith` (BOM:
composite 8000 + components 500) вместо привозного `habitat`. Объёмные стены — реголит-печать;
металл (через `components`) только в сосудах давления/машинах/точных узлах. **Различие объём/машина
— в BOM**: объёмные структуры тяготеют к `composite`, машинные (реактор/фаб/электролиз) — к
`components` (брешь №2b, `manufacturing-chain.md` §6).

**Радиация как новый механизм** (снимает отложенное D-048): хроническая доза даёт штраф к
`lifeExpectancy` (через демографию D-083) и/или к рождениям; построенная защита (`shield_berm`
или закапывание) штраф снимает. Острые дозы вспышек (`solar_flare`) уже есть — добавляется
хронический фон. Это движковый хук (см. эффекты ниже).

## P4 — Средства производства (частичная локализация ЗИП)

| id | icon | что делает | BOM | гейт |
|----|------|-----------|-----|------|
| `fab_shop` | 🏭🔩 | передел+фабрикация: металл → компоненты (литьё+аддитивка) | +components; −metals −энергия(много) −**chips**(немного, ЧПУ) | tech `fabrication` |
| `machine_shop` | ⚙🔧 | станки → ЗИП | +spares(локальные); −metals −энергия −**chips**(немного) | tech `machining` |

**`fab_shop` — материализация бреши №2b** (`manufacturing-chain.md`). Пока его нет, `components`
завозятся готовыми узлами (D-057); с `fab_shop` — производятся локально из `metals`. Марсианский
поворот честен: это аддитивно-литейный цех (энергия + чуть чипов на ЧПУ), НЕ земной прокатный
стан (тот массо-запретен к завозу). Фабрикационная мощность — новое узкое место темпа стройки.

**`spares` из чистого импорта → частично-локальный.** Механика: пока нет `machine_shop`, весь ЗИП
импортный (как сейчас). С `machine_shop` — доля `sparesLocalFrac` (например 0.7) покрывается
местным производством, остаток (прецизионный ЗИП) — импорт. Реально: точность/подшипники/датчики
требуют чипов и метрологии (чёрные узлы) → 100% локализовать нельзя. Пробивает пол импорта, но
не обнуляет (тезис цел). `fab_shop` и `machine_shop` — одна фабрикационная база (можно слить в
один объект, решается гриллингом).

Ограничитель по реальности: первые станки — импортная затравка; точность гейтнута метрологией
(чёрный узел graph.md `precision_metrology`).

## P6 — Кадры и образование (новый ресурс `specialists` + гейт)

| id | icon | что делает | BOM/эффект | гейт |
|----|------|-----------|------------|------|
| `school` | 🏫 | дети → квалификация | конвертит взросление в `specialists` | tech `education` |
| `university` | 🎓 | производит специалистов | +specialists; −энергия −opsCrew | tech `higher_education` |

**Квалификация как гейт хайтека:** продвинутые объекты (P8: `chip_fab`, `api_plant`) требуют не
только `minPop`, но и построенного пула `specialists` (новый гейт `minSpecialists`). Соединяет
демографию D-083 (дети — 7-оконная инвестиция) с D-028 (навыки MES-гейтнуты): без образования дети
дают только неквалифицированный труд.

Ограничитель по реальности: поколение времени на подготовку; семенные преподаватели — импорт.

**Рекомендация (анализ 2026-07-09): `specialists` — атрибут `Colonist`, а не стокапливаемый
ресурс-пул.** Пул — one-shot (построил университет, накопил стек, готово). Атрибут через
индивидуальную симуляцию D-083 даёт реальный кадровый цикл почти бесплатно: специалист учится
годы, стареет, болеет и умирает вместе с человеком; эпидемия бьёт не только по рукам, но и по
квалификации; bus factor (D-034) материализуется сам; образование нельзя выключить никогда.
Гейт `minSpecialists` тогда считает живых колонистов с атрибутом. Закрывает открытый вопрос
«ресурс-пул vs атрибут» из `implementation-roadmap.md`.

двигатель усиливает рождения при замкнутой еде/медицине/жилье/образовании.

Ограничитель по реальности: всё одновременно (энергия+вода+материалы+кадры); узкое место рвёт рост.

## Новые движковые эффекты (что реально трогаем в коде)

Scaffold уже имеет: `opsCrewMult`, `cureProb`, `lifeExpectancyBonus`, `repairRateMult`,
`unlockStructure`, `refuelStage3`. Нужны новые:

| эффект | что делает | расширяет | сложность |
|--------|-----------|-----------|-----------|
| `recycleBonus` | +η конкретному ресурсу (БЛСС) | resources.csv `recycle` фиксирован → делаем модифицируемым | средняя |
| `yieldMult` | ×выпуск структуры/класса | genMult-паттерн (D-075) | низкая |
| `radiationHazard` | хронич. доза → штраф lifeExpectancy/рождений; снимается защитой | новый механизм (снимает D-048) | **высокая** |
| `birthRateMult` | ×birthRate при условиях | D-083 births | низкая |
| `sparesLocalFrac` | доля ЗИП покрывается локально | previewOrder/spares (D-070 авто-ЗИП) | средняя |
| `minSpecialists` (гейт) | продвинутый объект требует пул специалистов | prereqMet (D-074 minPop) | средняя |
| `localOnly` (ресурс) | нельзя заказать с Земли | previewOrder/resources | низкая |
| `depletion` (структура) | накопленная добыча типа → рост цены энергии/труда за кг | второй дизайн-принцип §1; счётчик на тип | низкая-средняя |
| `rampYield` (структура) | выход нового процесса растёт с накопленным выпуском типа | второй дизайн-принцип §2; genMult-паттерн | низкая-средняя |

**Приоритет реализации эффектов:** сначала `unlockStructure`+`localOnly`+`yieldMult` (дёшево, дают
всю индустриальную лестницу P1–P4). `opsCrewMult` уже есть (P5). `recycleBonus`/`birthRateMult`/
`minSpecialists` — для P6–P7. `radiationHazard` — самый дорогой, отдельная фаза (может идти
параллельно как самостоятельный механизм, не блокируя лестницу).

## Структуры, которые НЕ надо механизировать (флейвор, не объект)

Чтобы не утонуть в структурах, часть реальных стадий — строка в BOM или переименование, не
отдельный объект:
- **Sabatier для топлива** — уже есть косвенно (`fuel`); отдельный объект не нужен, если топливо
  не критично для геймплея города.
- **Сепарация Ar** — флейвор, не моделируем.
- **Отдельные кирпич/серобетон/базальт** — свёрнуты в один `composite` (играбельность > точность).
- **Отдельные литьё/ковка/прокат** — свёрнуты в `foundry`+`machine_shop`.
- **Перхлораты** — НЕ отдельный тех «биоремедиация» (был бы one-shot) и не структура: постоянная
  строка BOM фермы на местном грунте (вода/энергия на промывку) — каждый новый гектар отмывается
  заново, реальный цикл в виде числа.

Правило: механизируем стадию, только если она добавляет РЕШЕНИЕ игроку (новый выбор/узкое место),
а не просто реалистичный шаг. Остальное — числа в BOM существующих цепочек.
