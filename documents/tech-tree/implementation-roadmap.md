# Инженерный роадмап — фазы реализации

> Как строить дерево технологий инкрементами. Каждая фаза — шиппабельный срез (D-номера, правки
> CSV/движка, тесты). Опора на каркас роадмапа-2 Блок C (`techs.ts`, `EarthOrder.unlockTech`,
> `techMods()`). Ordering снизу вверх по зависимостям; каждая фаза оставляет игру играбельной.

## Инженерные реалии кодовой базы (важно перед стартом)

- **Структуры — чистый CSV.** `STRUCTURES` грузится из `structures.csv`; добавить объект = строка.
  Дёшево. Вся индустриальная лестница P1–P7 — в основном новые строки structures.csv + гейты.
- **Ресурсы — НЕ чистый CSV.** Новый ресурс трогает `resources.ts` (`ResourceKind`, `RESOURCES`,
  колонки каталога), `colony-save.ts` (bump `SAVE_VERSION`, D-069 без миграции), иконки в UI
  (`colony-status`/`chronicle`/`earth-tab`), `previewOrder` (флаг `localOnly`). **Батчить добавление
  ресурсов по фазе**, чтобы амортизировать bump сейва и правки UI.
- **Техи — CSV + иногда код.** Строка в `techs.csv` дёшева; `unlockStructure` уже generic. Новые
  `TechEffect` (`recycleBonus`/`localOnly`/`radiationHazard`/…) требуют кода в `techMods()`.
- **Инвариант «пустой контент = прежнее поведение»** держится сам: объект не загружается, если его
  нет в CSV; тех ничего не гейтит, пока не куплен. Каждая фаза с пустым набором = зелёные старые тесты.

## P2 — Строительство из реголита + радиация (D-090 ядро реализовано 2026-07-09; радиация — открыт PR-5)

> **Статус: ядро сделано (D-090)** — `composite`/`components` (НЕ `localOnly`, дорого-но-возможно) +
> `sinter_plant`/`habitat_regolith`/`silo_regolith` под `regolith_construction`. **Радиация НЕ
> реализована** — сознательно вынесена в отдельный PR (сам этот файл ниже это и предлагал: «можно
> отдельным PR-треком»). Детали/альтернативы — `decisions.md` D-090; сводка — `index.md`.

Разблокирует объём жилья (обязательно для масштаба, `path-to-100k.md` §2) и снимает отложенное
радиационное давление (D-048).

- **Ресурс:** `composite` (местный объёмный стройматериал; спечённый реголит/серобетон/базальт
  свёрнуты в один, играбельность > точность).
- **Ресурс `components` (передел, брешь №2b — `manufacturing-chain.md`):** «здание из слитка» →
  вставить один слой передела. Структуры строятся из `composite` (объём) + `components` (узлы), а
  не из сырого steel/glass. Пока нет `fab_shop` (P3), `components` завозятся готовыми узлами
  (D-057) — плавный старт. Локальное производство `components` — в P3.
- **Структуры:** `sinter_plant` (regolith→composite), `habitat_regolith`/`silo_regolith` (BOM на
  `composite`+`components` вместо steel+glass), `shield_berm` (снимает радиационный штраф). Гейт —
  тех `regolith_construction`.
- **Радиация (новый механизм, высокая сложность — можно отдельным PR-треком):** хроническая доза →
  штраф к `lifeExpectancy` (демография D-083) и/или рождениям; снимается защитой (`shield_berm` или
  «закапывание»). Эффект `radiationHazard`. Острые дозы (`solar_flare`) уже есть.
- **Тесты:** объект из `composite` строится без стали; радиационный штраф применяется без защиты и
  снимается с ней; демография сдвигается корректно.
- **Майлстоун:** `local_construction`.

## P3 — Средства производства: передел + локальный ЗИП (D-091 ядро реализовано 2026-07-09)

> **Статус: ядро сделано (D-091).** `fab_shop`/`machine_shop` под ОДНИМ мерженым техом
> `fabrication` (план ниже разрешал слияние — взято). `sparesLocalFrac` НЕ реализован как отдельный

> Детали/альтернативы — `decisions.md` D-091; сводка — `index.md`.

Материализует брешь №2b (`components` локально) и пробивает пол импорта ЗИП частично (брешь №4).
Одна фабрикационная база.

- **Структуры:** `fab_shop` (metals→`components`, аддитивно-литейный цех; −энергия −немного chips
  на ЧПУ) — теперь `components` производятся локально, а не завозятся; `machine_shop` (→локальный
  `spares`). Гейты `fabrication`/`machining` (можно слить). Марсианский поворот: это НЕ земной
  прокатный стан (массо-запретен), а аддитивка/литьё (`manufacturing-chain.md` §4).
- **Эффект `sparesLocalFrac`:** доля ЗИП покрывается локально (напр. 0.7), остаток — импорт
  («прецизионный ЗИП», гейтнут метрологией/чипами). Правка `previewOrder`/авто-ЗИП (D-070): floor
  ЗИПа уменьшается на локальную долю.
- **Тесты:** `fab_shop` даёт `components` из `metals` (импорт узлов больше не нужен); с
  `machine_shop` требуемый импорт ЗИП падает на `sparesLocalFrac`; без них — как сейчас.
- **Майлстоуны:** `local_fabrication` (свои `components`), `local_spares`.

## Порядок и наименьший первый срез

**Зависимости:** P0 → P1 → (P2 ∥ P3 ∥ P4) → P5 → P6 → P7. Радиация — параллельный PR-трек после P0.
P2/P3/P4 независимы между собой (можно в любом порядке после P1).

**Рекомендация — начать с P0 + P1 одним заходом:**
- P0 делает дерево видимым/работающим (фундамент, ноль контента).
- P1 даёт немедленную, ощутимую отдачу (честная металлургия) и вводит паттерн «ресурс-батч +
  структуры + техи», который повторяют все следующие фазы.
- Вместе они — минимальный «доказательный» инкремент: игрок впервые видит дерево и локализует
  металлы. Всё остальное — повторение паттерна с новым контентом.

**Сложность по фазам:** P0 низкая · P1 средняя (первый ресурс-батч) · P2 высокая (радиация) ·
P3 средняя · P4 низкая · P5 средняя · P6 средняя-высокая · P7 средняя (в основном CSV+баланс).

## Общие правила реализации (каждая фаза)

1. Ресурсы — батчем на фазу (амортизировать bump сейва D-069 + иконки UI).
2. Каждая фаза: D-номер(а) в `decisions.md` ПЕРЕД кодом; строка в лог `index.md`; счётчик тестов.
3. Инвариант: пустой набор фазы = прежнее поведение (зелёные старые тесты без правок — если
   правка нужна, понять почему, а не подгонять).
4. Числа TOY, калибровка плейтестами (как resources.csv); порядки — из `gaps-vs-reality.md`/
   `real-industrial-stages.md`/`references.md`.
5. Отдельный `describe` на фазу в тестах; CLI-смоук на новую цепочку.
6. Не механизировать реальную стадию, если она не добавляет РЕШЕНИЯ игроку (см. `content-proposal.md`
   «что не механизировать»).
