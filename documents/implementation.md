# Реализация Outland — архитектура и план

> План кодовой базы. Дизайн-истина — в `mechanics.md`/`decisions.md`; здесь — *как строим*.
> Статус: проектирование (вертикальный срез). Решение стека — D-040.

---

## 1. Стек и принцип (D-040)

- **Платформа:** веб, **TypeScript**. Игра-аргумент должна легко шариться (тезис доходит до людей).
- **Несущий принцип — разделение движка и UI:**
  - **Движок (`src/engine/`)** — **чистый TS, ноль UI-зависимостей.** Это и есть аргумент игры
    (честный расчёт, §7.1) → должен быть полностью юнит-тестируем и детерминирован (сид-RNG).
    Портирует `prototype/economy.py` как референс.
  - **UI (`src/ui/`)** — тонкий слой над состоянием движка на **Lit** (веб-компоненты, ~5КБ,
    стандартные Custom Elements, реактивный рендер по свойствам). Легче полноценного фреймворка,
    но структурнее vanilla: панели §8 — самостоятельные компоненты. Игра пошаговая (перерисовка
    редкая, на коммит), состояние централизовано в движке. Решение изолировано разделением движок ⟂ UI.
- **Сборка/тесты:** Vite + Vitest. Сейв/лоад — JSON-состояние в localStorage.

## 2. Слои движка (`src/engine/`)

| Модуль | Содержимое | Источник |
|--------|------------|----------|
| `graph.ts` | граф 27 узлов как данные (Node[]) | `documents/graph.md` |
| `types.ts` | `Node`, `Params`, `GameState`, статусы узлов | economy.py dataclasses |
| `sim.ts` | чистые функции: `needs`, `importBreakdown`, `autonomyByMass`, `survivalRunway`, `step` | economy.py |
| `rng.ts` | сид-генератор (детерминизм для реплея/тестов, D-011) | economy.py random |
| `policy.ts` | жадная авто-политика — **baseline-ИИ и оракул для тестов**, не игрок | economy.py greedy |

**Золотые тесты:** движок воспроизводит находки прототипа на калиброванных числах — автономия
плато 67–77%, **запас хода 0.5 в каждом окне**, эфф. $/кг ~$1M (D-037/D-038). Если число съехало —
сломали модель.

## 3. Слой UI (`src/ui/`) — отображает §8

Хаб-дашборд + спицы (D-018). Каждый — **Lit-компонент** (Custom Element), 1:1 с §8:

- `Dashboard` — дом: автономия **громко**, `F`/счёт за импорт **тускло**, ресурсы, люди+bus factor,
  «в пути», объекты со статусами 🟢🟡🔴⚫, окно/год, субсидия `M`. **Самодостаточность НЕ показывается** (D-025).
- `ObjectTree` — drill-down BOM: make-or-buy на узле, оптимистичная вилка цены (MES скрыт, D-014),
  туман `?` на неразведанном.
- `ImportPanel` — что импортируем, кг/окно, вклад в `F`, влияние на `M`.
- `WindowManifest` — распределение `M` (F обязателен первым) → коммит → ход времени.
- `Debrief` — кривые в ретроспективе + впервые названный **запас хода** (D-025), диегетически, без морали (D-036).

**Состояние/действия:** одно `GameState`; действия — explore / allocate (localize · import · colonists ·
launch-capacity `K`) / commit. Время идёт **только на коммите** (синодическое окно, §7.2).

## 4. Объём вертикального среза

**В срезе (одна играбельная петля наскозь, §8.1):**
- старт → игра → одна из концовок (коллапс / штиль / отмена субсидии, §7.4);
- дашборд с ключевыми метриками; drill-down make-or-buy хотя бы по узлу; манифест+коммит+ход; дебриф;
- движок целиком: 27 узлов как данные, единый денежный ledger, производный спрос, MES-гейты, износ,
  базовые события, рычаг населения, пусковая мощность `K` (D-038, одна ручка).

**Отложено (за срезом):**
- лестница технологий запуска Gen0→Gen1 (D-039) — пока одна ручка `K`;
- платная разведка/гранулярность тумана (D-035) — упрощённо;
- пулы специалистов детально (D-034) — bus factor минимально; мегапроекты (D-033) — заглушка;
- полировка нарратива/арта, реиграбельные старты (§7.6).

## 5. Пошаговый план сборки

Фазы упорядочены так, что **срез играбелен уже после Фазы 3**; дальше — углубление.

- **Фаза 0 — Скаффолд.** Vite (vanilla-ts) + Lit + Vitest; `tsconfig` strict; конфиги; структура
  `src/engine/` ⟂ `src/ui/`; пустой `outland-app`.
  *Готово:* `npm run dev` поднимает страницу, `npm test` зелёный.
- **Фаза 1 — Порт движка (TDD).** `types.ts` (Node, Params, GameState), `graph.ts` (27 узлов),
  `rng.ts` (сид-генератор), `sim.ts` (mes, tailFrac, needs, importBreakdown, launchMaint,
  autonomyByMass, survivalRunway, step), `policy.ts` (жадная авто-политика). ⚠️ Python `random`
  точно не воспроизвести → золотые тесты: events-OFF — точные числа, events-ON — статистика по сидам.
  *Готово:* тесты воспроизводят прототип — автономия плато 67–78%, запас хода 0.5 каждое окно, эфф. $/кг ~$1M.
- **Фаза 2 — Минимальный дашборд + «Ход».** `store.ts` (GameState + step + pub/sub),
  `dashboard-panel.ts` (автономия громко, F/M тускло, население, окно/год, статусы 🟢🟡🔴⚫;
  самодостаточность НЕ показывается).
  *Готово:* кликаешь окна — видишь плато автономии и подъём F.
- **Фаза 3 — Распределение бюджета** ← *срез играбелен*. Рефактор `step()` под решение игрока
  (что локализовать, мощность `K`, колонисты, импорт = остаток); `window-manifest.ts` (деление `M`,
  F первым, → коммит → ход). Жадная политика остаётся для тестов/ИИ.
  *Готово:* проходишь партию руками от старта.
- **Фаза 4 — Дерево объекта.** `object-tree.ts` (BOM-дерево, make-or-buy на узле, оптимистичная
  вилка цены с скрытым MES, туман `?`), `import-panel.ts` (деталь импорта, кг/окно, вклад в F).
  *Готово:* ныряешь в узел, видишь рецепт, решаешь make/buy.
- **Фаза 5 — Концовки + дебриф.** Детект конца (коллапс / штиль / отмена субсидии, §7.4);
  `debrief-panel.ts` (кривые, потолок чёрных узлов, впервые названный запас хода D-025, диегетически).
  *Готово:* партия доходит до концовки, дебриф называет запас хода.
- **Фаза 6 — Сейв/лоад.** Сериализация `GameState` в localStorage; загрузка/новая игра.
  *Готово:* перезагрузка браузера сохраняет состояние.

Срез играбелен уже после шага 4; шаги 5–7 углубляют.

## 6. Открытые вопросы по реализации

- UI на Lit: состояние — внешний стор/синглтон движка, компоненты подписываются и зовут действия;
  уточнить паттерн связи (reactive controller vs простой pub/sub) на скаффолде. Низкий риск — движок изолирован.
- Где живут данные графа: `.ts`-литерал (типобезопасно) vs `.json` (правится без сборки). Раб. оценка — `.ts`.
- Нужен ли движку веб-воркер (тяжёлый `step`)? Вряд ли — шаг дёшев, пересчёт раз в коммит.
- **Тулчейн: старт на стандартном Vite + Vitest.** Vite+ (унифицированный тулчейн VoidZero,
  Vite/Vitest/Rolldown/Oxc в одном) — **отложенный кандидат**: очень свежий и только что куплен
  Cloudflare (03.06.2026, лицензия/будущее в движении), а обычный Vite уже покрывает нужды среза.
  Пересмотреть, когда стабилизируется после поглощения — свап дёшев (надстройка над тем же Vite).
