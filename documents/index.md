# Outland — контекст проекта

> Корневой контекст-файл. Точка входа для любой новой сессии. Читать первым.
> Статус документа: **черновик** (навык `project-context` ещё проектируется).

## Питч / центральный тезис

**Outland** — **симуляция с tycoon-элементами** (D-046): игрок строит и держит марсианскую колонию
на *пульсирующем снабжении* (синодические окна ~26 мес). Устойчивая **частично-автономная** колония
**достижима**: местный балк (вода/O₂/еда/топливо/стройматериалы) на 100% + земной хайтек (чипы, фарма,
катализаторы — конечный, но недостижимый при колониальном масштабе MES, D-045) завозом, с буферами X+2.

Игра — про **инженерию устойчивости**: замкнуть что можно, импортировать что нельзя, пережить
деградацию/утечки/кризисы и остаться стойким к пропуску окон. Интеллектуальная глубина (MES,
незамыкаемые циклы, дороговизна, Циолковский, износ) — это **структура сложности/цены**, не рок.
Полная независимость от Земли — трудная поздняя цель (финал-босс).

> **Поворот 2026-06-19 (D-046):** прежний тезис «устойчивая автономия практически *невозможна*»
> (трагедия/игра-аргумент) **переосмыслен** в «трудно, дорого, но достижимо — и в этом игра».
> Старая рамка — в `mechanics.md` (срез v1); активный дизайн v2 — `colony-sim.md`.

## Ключевые механики (дайджест; детали → mechanics.md)

1. **Граф зависимостей** — рекурсивное дерево: один насос разворачивается в сотни узлов.
2. **Тирания Циолковского + синодические окна** — масса как валюта, запуск раз в ~26 мес.
3. **Асимптота замыкания** — последние % автономии экспоненциально дороги; часть недостижима.
4. **Энтропия и износ** — замкнутые системы текут; вечный пол импорта на запчасти.
5. **Хрупкость малой популяции** — нет специалиста под каждую систему; потеря критична.

## Референсы для калибровки (дайджест; детали → references.md)

- **Биосфера-2** — провал по кислороду; пример незамкнутости.
- **БИОС-3 / MELiSSA / NASA ECLSS** — замкнутые системы жизнеобеспечения.
- **ISRU** — Sabatier, MOXIE; добыча ресурсов на месте.
- **Mars Direct (Зубрин)** — оптимистичный контртезис.
- **Минимальная жизнеспособная популяция** — литература по порогам.

- **D-005** Иерархия — 5 файлов, под-документы создаются лениво, глоссарий внутри `mechanics.md`.
- **D-007** Состав `index.md` — 6 секций + 2 дайджеста (механики, референсы) по строке на пункт.
- **D-008** Триггеры навыка — авто-контекст по `index.md` + явные фразы + границы сессии + создание с нуля.
- **D-009** Ядро игры — граф зависимостей на неравенстве «спрос ≥ MES» с агрегацией спроса вверх по дереву.
- **D-010** Темп прозрения — три акта (соблазн→сомнение→прозрение), игрока соблазняет автономия %, беда растёт тускло.
- **D-011** Импорт-эскейп на каждом узле + раскрытие спроса сверху вниз («импортируй — или осознай глубину»).
- **D-012** Двигатель ходов — синодические окна (~26 мес) + масса-валюта (Циолковский); зажим F vs капитал.
- **D-013** Износ — утечка циклов (η) + деградация капитала; хвост обслуживания растит F с задержкой.
- **D-014** MES скрыт от игрока, выводится эмпирически по реализованной удельной цене (статусы 🟢/🟡 обнаруживаемы).
- **D-015** Хрупкость популяции — у навыков тоже есть MES; вечный импорт экспертизы, единые точки отказа.
- **D-016** Жанр/loop — пошаговая эконом-стратегия; петля окна: обзор→план→бюджет→коммит→ход времени.
- **D-017** Победа переосмыслена — трагедия, а не триумф; концовки коллапс/отмена/штиль + дебриф-прозрение.
- **D-018** UI — хаб-дашборд + спицы; граф как drill-down «дерево объекта» (BOM), панель импорта, манифест окна.
- **D-019** Стартовый набор чёрных узлов — чипы и катализаторы как корневые отравители + метрология/фарма/спецсплавы.
- **D-020** Чёрный ≠ «нет элемента»: Марс даёт C/H/O/N; стена — MES + катализаторы. Полимеры/эпоксиды — флагман.

- **D-076** Тест-плей после деплоя показал субсидию `M=$20B` жёсткой стеной: расчёт по окнам — 20 колонистов съедают 100% бюджета к окну 30, R&D ст.1 дозаправки превышает 100% уже к окну 10. [баланс, экономика]
- **D-077** Дыра: весь наземный R&D дозаправки покупался ДО отправки первых колонистов — проверялся только прогресс лестницы, населения в проверке не было. Новая `rndOk = !order.unlockRefuel || s.everHadPop` складывается в `feasible` наравне с `prereqsOk` — попытка без высадки делает весь заказ нефизибельным (деньги не списываются), а не тихо промахивается мимо одной строки.
