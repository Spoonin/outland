## D-001 — Артефакт = процессный навык `project-context`   (2026-06-13, статус: accepted)
**Контекст:** нужно решить, что именно мы строим в первую очередь.
**Решение:** строим процессный навык — свод правил, по которым Claude ведёт живой контекст проекта. Outland — первый проект-применение и тестовый пример.
**Альтернативы:** методичка по геймдизайну (отклонено — не переиспользуема); статичная обёртка вокруг одной игры (отклонено — не масштабируется на другие проекты).

## D-002 — Контекст — живой документ между сессиями   (2026-06-13, статус: accepted)
**Контекст:** боль — каждый новый чат начинается с нуля, проект приходится пересказывать.
**Решение:** навык отвечает и за создание, и за поддержание актуальности контекста между сессиями. Ценность — «передача эстафеты».
**Альтернативы:** разовая генерация документа без сопровождения (отклонено — быстро устаревает).

**Контекст:** где хранить канонический контекст.
**Решение:** локальный файл в git-репозитории: `documents/index.md` (корневое оглавление) + иерархия под-документов рядом.

## D-004 — Основная среда — Claude Code   (2026-06-13, статус: accepted)

## D-005 — Структура иерархии под-документов   (2026-06-13, статус: accepted)
**Контекст:** какой набор файлов держать рядом с `index.md` (Узел 5).
**Решение:** 5 файлов — `index.md` (карта+питч+статус+своды), `decisions.md` (этот журнал), `open-questions.md`, `mechanics.md`, `references.md`. Под-документы создаются **лениво** — файл появляется на диске только при наличии реального содержимого; до этого помечен «не создан» в карте `index.md`. Глоссарий — секцией внутри `mechanics.md`, отдельный `glossary.md` заводим только при разрастании (см. будущий узел бюджета объёма).
**Альтернативы:** сразу заводить пустые плейсхолдеры (отклонено — разрастание пустышек); отдельный `glossary.md` с самого начала (отложено до необходимости).

## D-007 — Состав секций `index.md`   (2026-06-13, статус: accepted)
**Контекст:** какой финальный набор секций держать в корневом файле (Узел 6).
**Решение:** 6 основных секций — Питч/тезис, Статус, Карта документов, Свод ключевых решений, Топ открытых вопросов, Лог изменений — плюс 2 дайджест-секции (Ключевые механики, Референсы), где допускается строго одна строка на пункт со ссылкой на под-документ. Дайджесты оставлены ради самодостаточности входа (D-002, «передача эстафеты»): новая сессия понимает суть проекта, прочитав только `index.md`.
**Альтернативы:** чисто навигационный index без механик/референсов (отклонено — заставляет открывать под-документы только чтобы понять, о чём проект); полные секции механик/референсов в index (отклонено — дублирование с под-документами, гарантированный рассинхрон, разрастание против бюджета объёма).

## D-008 — Триггеры навыка   (2026-06-13, статус: accepted)
**Контекст:** при каких сигналах срабатывает навык; что писать в `description` SKILL.md (Узел 7).
**Решение:** три класса триггеров — (1) контекстный/автоматический: репо с `documents/index.md` → читать первым; (2) явные фразы («обнови контекст», «зафиксируй решение», «продолжаем проект», «подхвати где остановились», «заверши сессию»); (3) границы сессии (старт — прочитать+свериться, конец — синхронизировать+предложить коммит). Плюс режим создания с нуля по явной фразе; само отсутствие `index.md` навык НЕ активирует. Заодно починен фронтматтер SKILL.md (нормальный YAML `name`/`description`), тело подтянуто под D-005/D-007.
**Альтернативы:** активировать навык в любом репо без `index.md` (отклонено — навык лез бы в каждый проект); только явные фразы без авто-контекста (отклонено — теряется главная ценность «передачи эстафеты»).

## D-009 — Ядро игры = граф зависимостей на «спрос ≥ MES»   (2026-06-13, статус: accepted)
**Контекст:** как превратить тезис о незамыкаемых циклах в работающую механику, а не лозунг (детали → `mechanics.md` §1).
**Решение:** ядро — граф зависимостей, где у каждого узла есть минимальный эффективный масштаб (MES) и спрос колонии D. Узел эффективно локализуем при `D ≥ MES` + локализованном поддереве. Спрос агрегируется вверх по дереву (общий апстрим кормится суммой потребителей), но ограничен населением → положительная обратная связь упирается в потолок → асимптота замыкания, последний % недостижим в принципе.
**Альтернативы:** чистый tech-tree без экономики масштаба (отклонено — не доказывает тезис, автономия выглядела бы достижимой); скриптовый «проигрыш по сюжету» (отклонено — игрок должен сам упереться в стену, иначе тезис не прочувствуется).

## D-010 — Темп прозрения: соблазн ложной метрикой   (2026-06-13, статус: accepted)
**Контекст:** ключевое балансное требование — игрок должен осознать схождение к провалу постепенно, не моментально и не внезапным обрывом (детали → `mechanics.md` §2).
**Решение:** трёхактная кривая (соблазн → сомнение → прозрение), эмерджентная из чисел. Игрока соблазняет ярко показанная автономия %, тогда как настоящие метрики (пол импорта F, стоимость закрытия следующего %) растут тускло и с задержкой. Балансные ручки: экспонента MES по тирам (k), скорость износа → наклон F, бюджет массы окна B vs рост спроса, глубина/число чёрных узлов, график раскрытия информации. Избегаем двух ошибок: «слишком очевидно» (рельсы) и «слишком скрыто» (нечестный обрыв).
**Альтернативы:** мгновенно честная подача всех метрик (отклонено — нет игры, нет соблазна); скриптовые акты по таймеру (отклонено — должно следовать из чисел, иначе ощущается как рельсы).

## D-011 — Импорт-эскейп + раскрытие сверху вниз   (2026-06-13, статус: accepted)
**Контекст:** как сделать раскрытие дерева постепенным, если производство кажется обязанным идти снизу вверх (детали → `mechanics.md` §1.2, §3). Слоган: «либо импортируешь компонент — либо осознаёшь всю глубину своей проблемы».
**Решение:** дерево детерминировано (на нём держится баланс), недетерминировано только знание игрока (туман) + события + оценки. Импорт — всегда доступный вход на каждом узле, поэтому локализация требует лишь доступных ПРЯМЫХ входов (локально ИЛИ импортом); строить снизу вверх обязателен только полностью локализуемый кусок. Спрос разворачивается сверху вниз тир за тиром по решениям make-or-buy — это и есть раскрытие тумана. Чёрный узел вскрывается поздно, т.к. до погони за последними % не было причины копать так глубоко. Оценки скрытых узлов смещены в оптимизм и уточняются разведкой — это склеивает недетерминированность с темпом прозрения (D-010).
**Альтернативы:** процедурно-случайная структура дерева (отклонено — нельзя гарантировать схождение к провалу и баланс §2); строго снизу вверх без импорт-эскейпа (отклонено — делает постепенное раскрытие логически невозможным).

## D-012 — Двигатель ходов = синодические окна + масса-валюта   (2026-06-13, статус: accepted)
**Контекст:** что превращает статичный граф в игру во времени и откуда берётся бюджет массы B (детали → `mechanics.md` §4).
**Решение:** ход = одно синодическое окно (~26 мес, партия ~25–40 окон). Дефицитная валюта — масса на грунт Марса (тирания Циолковского: уравнение ракеты). Каждое окно игрок делит бюджет B между обязательным импорт-floor F, капиталом (автономия) и людьми; F забирает B первым → по мере F→B свободный капитал→0 (механически Акт 3 из §2). Лаг 1–3 хода усиливает ловушку оптимизма (D-010/D-011). Леверы массы (ISRU/локальное топливо) — узлы графа → двигатель и граф сцеплены. Крупнозернистое время + ратчет населения → хрупкость.
**Альтернативы:** непрерывное/реальное время (отклонено — синодические окна дают крупный масштаб и медленное прозрение, физически честны); масса как абстрактные «кредиты» без уравнения ракеты (отклонено — теряется тирания Циолковского, ключевой источник дороговизны).

## D-013 — Износ как двигатель роста пола импорта   (2026-06-13, статус: accepted)
**Контекст:** что порождает и растит пол импорта F и держит узлы привязанными к импорту (детали → `mechanics.md` §5).
**Решение:** два рода износа — утечка вещества в замкнутых циклах (КПД рециклинга η<100% → вечная подпитка) и деградация капитала (MTBF → запчасти → замена). Каждый локализованный узел несёт хвост обслуживания (запчасти+подпитка); локализация снижает основной импорт, но добавляет меньший импорт обслуживания — ни одна ветка не обнуляется. Хвост рекурсирует вниз как производство → погоня за нулём упёка загоняет в чёрные узлы. Износ копится по окнам → F растёт с задержкой (топливо для прозрения, D-010). η-асимптота = механика «асимптоты замыкания». Отложенное обслуживание → каскадные отказы (хрупкость).
**Альтернативы:** локализованные узлы бесплатны навсегда (отклонено — F не растёт, нет зажима, нет тезиса); износ как чисто случайные поломки без хвоста обслуживания (отклонено — теряется связь «локализация оставляет импортный остаток» и рекурсия глубины).

## D-014 — MES скрыт, выводится игроком эмпирически   (2026-06-13, статус: accepted)
**Контекст:** показывать ли игроку MES числом (детали → `mechanics.md` §1.3, §1.6).
**Решение:** MES детерминирован в модели (баланс держится, D-011), но игроку НЕ показывается. Игрок наблюдает только реализованную удельную цену после постройки и выводит порог эмпирически — пробуя масштабы и агрегируя спрос (как сэмплируя скрытую функцию). Статусы 🟢/🟡 обнаруживаемые, а не подписанные: эффективен узел или субмасштабен, выясняется по эксплуатации. До-постройочная оценка оптимистично смещена (§3.4) → субмасштабный завод иногда яма, в которую масса уже вложена (лаг §4.4).
**Альтернативы:** показывать MES числом (отклонено — игрок считает выгоду до постройки, рушится соблазн D-010 и туман D-011); полностью случайный MES (отклонено — ломает баланс; скрытость ≠ случайность).

## D-015 — Хрупкость популяции: у навыков тоже есть MES   (2026-06-13, статус: accepted)
**Контекст:** как смоделировать малое население как источник хрупкости, не как отдельную подсистему (детали → `mechanics.md` §6).
**Решение:** население двойственно — источник спроса D (растит F) и источник труда/навыков (узлы надо эксплуатировать). Узлы требуют специалистов, не только капитал. Ключевое: у навыков тоже есть MES — локальное обучение специальности требует учебной базы с минимальным масштабом; колония ниже порога импортирует экспертизу навсегда (человеческий пол импорта, структурно тот же F). Хрупкость = единые точки отказа (соблазн держать 1 специалиста без резерва); потеря гасит узел → каскад (§5.6), знание теряется безвозвратно, если не передано. Тезис незамыкаемости распространён на кадры.
**Альтернативы:** население как простой множитель труда без специальностей (отклонено — теряется хрупкость и «человеческий пол импорта»); генетическая минимальная популяция как главный фокус (отклонено — игра про замыкание циклов, а не про демографию; порог навыков важнее).

## D-016 — Жанр и game loop   (2026-06-13, статус: accepted)
**Контекст:** что игрок физически делает каждый ход; как модель становится игрой (детали → `mechanics.md` §7).
**Решение:** пошаговая экономическая стратегия / менеджмент колонии, базовый глагол make-or-buy по скрытому графу под бюджетом массы; платформа — компьютер. Петля окна: обзор → разведка/план → распределение B → коммит → ход времени (~2,2 года) с запаздывающими последствиями. Ритм «планируй в неопределённости → коммить массу → жди → узнавай последствия». Поверхностная цель — автономия→100%, реально недостижимая (соблазн).
**Альтернативы:** реальное время / Factorio-подобный автоматизатор (отклонено — теряется крупный масштаб окон и честный расчёт); PvP/конкуренция (отклонено — противник игры — физика, не оппонент).

## D-017 — Победа переосмыслена: трагедия, а не триумф   (2026-06-13, статус: accepted)
**Контекст:** что значит «выиграть» в игре, доказывающей невозможность автономии (детали → `mechanics.md` §7.4–7.6).
**Решение:** триумфальной победы-автономии нет. Концовки: коллапс / отмена проекта Землёй (импорт неподъёмен) / асимптотический штиль (выживание при вечной зависимости). Лучший исход — стабильная, но вечно зависимая колония, что само доказывает тезис. Финальный дебриф показывает кривую в ретроспективе (плато автономии, F, чёрные узлы) — «заслуженное прозрение» (§2.5). Реиграбельность подтверждает тезис: разные старты → стена в разных местах.
**Альтернативы:** классическая победа при 100% автономии (отклонено — прямо противоречит тезису); чистый невыигрываемый софтлок без дебрифа (отклонено — игрок почувствует обман, а не прозрение, ср. §2.5).

## D-018 — UI: хаб-дашборд + спицы, граф как drill-down   (2026-06-13, статус: accepted)
**Контекст:** как игрок видит и трогает модель; представление «доски»-графа (детали → `mechanics.md` §8).
**Решение:** архитектура хаб-дашборд + спицы. Дашборд (дом) — ресурсы, люди/специалисты, грузы и люди в пути, состояния инфраструктуры/объектов, громкая автономия %, тусклые F/счёт импорта. Граф НЕ постоянный фон, а контекстный drill-down «дерево объекта» (отступ-BOM) при добавлении/локализации; отдельная спица «панель импорта» при решении buy; экран «манифест окна» для распределения B. Дерево показывает make-or-buy, оптимистичную оценку с вилкой (MES скрыт, D-014), специалистов/bus-factor (D-015), туман с [разведать]. Подача: честное недоговаривание (не god-mode), асимметрия заметности (хорошее громко / плохое тихо, D-010), drill-down требует усилия → не-заглядывание готовит прозрение.
**Альтернативы:** постоянный полноэкранный граф как главный вид (отклонено — давит, плохо масштабируется на сотни узлов); чистый node-link граф вместо BOM (отклонено — хуже масштаб; фокус-граф оставлен как опция для агрегации, §8.8).

## D-019 — Стартовый набор чёрных узлов   (2026-06-13, статус: accepted)
**Контекст:** контент — что именно недостижимо для колонии и почему (детали → `mechanics.md` §9).
**Решение:** старт из 5 чёрных узлов: передовые чипы, прецизионная метрология/эталоны, спецкатализаторы, сложная фарма/биопрепараты, спецсплавы/монокристаллы. Два «корневых отравителя» (чипы и катализаторы) пронизывают почти весь граф и гарантируют асимптоту; остальные дают глубину. Коварный приём: катализаторы отравляют ISRU/топливо — рычаг удешевления импорта (§4.5) сам упирается в чёрный узел. У каждого зафиксированы: почему чёрный, что отравляет вниз, путь к озарению от бытового товара. Число/глубина узлов — рычаг темпа (§2.4); точные MES/спрос — калибровка в `references.md`.
**Альтернативы:** один чёрный узел (отклонено — мало охвата, тезис выглядел бы частным случаем); десятки разрозненных (отклонено — размывает; лучше 2 корневых отравителя + опора).

## D-020 — «Чёрный ≠ нет элемента»; полимеры/эпоксиды как флагман   (2026-06-13, статус: accepted)
**Контекст:** вопрос — есть ли на Марсе углеводороды для полимеров/эпоксидов; уточнение природы чёрного узла (детали → `mechanics.md` §9.6, `references.md` §1).
**Решение:** чёрный узел — это не отсутствие элемента. Марс даёт C/H/O/N (CO₂-атмосфера, водяной лёд, N₂/Ar); ископаемых углеводородов нет, но они синтезируются (Sabatier, RWGS+Фишер-Тропш → олефины → ПЭ/ПП, этиленоксид/эпоксиды). Стена — MES синтез-цепочек + зависимость от катализаторов (корневой узел 9.3). Полимеры/эпоксиды — флагманский пример жёлто-красной цепочки с катализаторным хвостом импорта (ПЭ/ПП 🟡; ароматические эпоксиды ≈⚫ на масштабе колонии). Заведён `references.md` с марсианскими ISRU-ресурсами как калибровочной базой.
**Альтернативы:** делать сырьё/элементы дефицитом как источник сложности (отклонено — подменяет тезис «незамкнутость циклов» на «нет ресурсов», что игра как раз опровергает).

## D-021 — Старт с нуля и денежный бюджет 1 трлн/синод   (2026-06-13, статус: accepted)
**Контекст:** с чего колония начинает и в чём измеряется бюджет игрока (детали → `mechanics.md` §10, уточнение §4.2).
**Решение:** колония стартует с нуля (всё импорт, автономия 0%). Бюджет ДЕНЕЖНЫЙ: каждый синод Земля даёт субсидию M≈1 трлн валюты (вымышленной, эквивалент USD) на ракеты с грузами/людьми. Деньги → масса через стоимость кг-на-грунт c (тирания Циолковского): триллион покупает обидно мало (шок ценника как механика). M ~фиксирован и НЕ растёт за вложения; вложения (носители, ISRU) снижают c. F в деньгах растёт с населением/износом и съедает триллион → асимптота. Концовка «отмена» = Земля обрезает субсидию.
**Альтернативы:** бюджет в массе напрямую без денег (отклонено — теряется «шок ценника» и связь с реальной дороговизной); растущий со временем бюджет (отклонено — снимает зажим, противоречит асимптоте).

## D-022 — Мегапроекты / «ложные спасители»   (2026-06-13, статус: accepted)
**Контекст:** идея — термояд как дорогой прорыв, который откладывает коллапс, но всё равно упирается в импорт (детали → `mechanics.md` §11).
**Решение:** класс объектов «ложный спаситель» — огромный дорогой импорт, релаксирующий ограничитель, который НЕ был настоящим узким местом (энергия, стоимость запуска c, жильё), и НИКОГДА — замыкание (MES/чёрные узлы). Правило: задержка, а не решение → потолок автономии сдвигается, асимптота держится. Флагман — термоядерная установка: дешёвая энергия удешевляет эксплуатацию, но сам реактор зависит от чёрных узлов (сверхпроводники/чипы) и добавляет к вечному F. Воплощает Акт 2 («нужна одна технология»): задержка → стена усиливает прозрение. Другие кандидаты: дешёвый запуск, реактор деления, «универсальный фабрикатор».
**Альтернативы:** мегапроект, реально решающий автономию (отклонено — ломает тезис); отсутствие мегапроектов (отклонено — теряется осязаемое искушение Акта 2, самый сильный момент ложной надежды).

## D-023 — Игрок наполняет завоз сам; прайс-каталог цена=земная+масса×c   (2026-06-13, статус: accepted)
**Контекст:** стартовый завоз — пресет или игрок сам? как устроены цены (детали → `mechanics.md` §10.1, §10.3).
**Решение:** пресета нет — игрок с первого хода собирает манифесты из прайс-каталога под бюджет M. Цена = земная_стоимость + масса×c. Один курс делает работу тезиса: дёшево-тяжёлое (вода, сталь, балк) доминируется доставкой → выгодно локализовать (низко-MES, лёгкие победы Акта 1); высокотехнологичное (чипы/фарма/катализаторы) лёгкое и дорогое по сути → импорт вечен. Экономика сама ведёт игрока локализовать балк и гордиться автономией %, пока F сидит в хайтеке. Если автономия считается по массе — эффект усиливается (метрика массы как соблазн D-010). Авторская трагедия → заслуженное прозрение (§2.5).
**Альтернативы:** скриптовый пресет стартовой колонии (отклонено — теряется шок ценника и авторство трагедии); цена без компоненты масса×c (отклонено — пропадает экономический градиент «локализуй балк / импортируй хайтек», несущий ловушку).
**Уточнено D-038/D-039:** компонента доставки в цене — больше не курс `c`, а капитал пусковой мощности (топливо·масса + содержание `K`); градиент «локализуй балк / импортируй хайтек» сохраняется, источник дороговизны уточнён.

## D-024 — События: стохастический слой, вероятности от состояния   (2026-06-13, статус: accepted)
**Контекст:** какие события и как они вписаны, чтобы не быть произволом (детали → `mechanics.md` §12).
**Решение:** случайность поверх детерминированной истины (слой 3, D-011); вероятности задаёт состояние колонии (тонкие запасы, отложенное обслуживание, единые точки отказа, ненадёжные пуски) → беда бьёт там, где срезаны углы (§2.5). Категории: аварии при запуске (деньги потрачены, масса не доставлена + лаг), аварии в пути (потеря между окнами), поломки на Марсе (износ→отказ→каскад §5.6), смертность от еды и фармы (недобор рациона/медпокрытия → смерти → потеря специалистов → спираль коллапса, механизм концовки D-017). Размен резерв vs риск: буферы стоят бюджета сейчас, лин-режим дешевле и опаснее (ловушка оптимизма D-010).
**Альтернативы:** чисто случайные события без связи с состоянием (отклонено — произвол, ощущается нечестно); полностью детерминированная игра без событий (отклонено — нет напряжения, нет хрупкости §6, нет реиграбельности).

## D-025 — Две метрики: автономия vs самодостаточность   (2026-06-13, статус: accepted, грилл)
**Контекст:** «автономия» была перегружена («масса или ценность») и служила и целью, и правдой одновременно (уточняет D-009/D-010, §1.5, §7).
**Решение:** разнести на два термина. **Автономия** — доля спроса по массе, локализованная локально; соблазнительный заголовок (растёт быстро, т.к. балк тяжёлый). **Самодостаточность** — пережила бы колония обрыв импорта; измеряется как **запас хода в окнах** (симуляция спирали §12.5 при импорте=0), под капотом — критичность-взвешенное покрытие. Самодостаточность **дебриф-онли**: не живой гейдж, выводима в игре из `F` и цветов узлов, явно названа (как runway) только в дебрифе. Глоссарий: `CONTEXT.md`.
**Альтернативы:** одна метрика (отклонено — либо лжёт, либо убивает соблазн); самодостаточность как процент (отклонено — абстрактна; «через 14 мес все мертвы» бьёт сильнее).

## D-026 — Структура: курируемый граф авторских узлов   (2026-06-13, статус: accepted, грилл)
**Контекст:** §1 описывал граф узлов, прототип — тир-лестницу, UI — «объект» (уточняет D-009).
**Решение:** игре-фейсинг истина — **граф индивидуально-авторских узлов** (несколько десятков, не сотни), «узел» — каноничный термин, «объект» в UI = узел. **«Тир» понижен до атрибута глубины** узла, задающего MES. Тир-агрегация прототипа была расчётным шорткатом для проверки кривой, не игрой.
**Альтернативы:** тир-лестница как настоящая игра (отклонено — убивает «прослеживание насоса до чёрного узла», §9); процедурный граф (отклонено — нельзя гарантировать баланс/тезис, D-011).

## D-027 — Единый денежный ledger; масса-бюджет B упразднён   (2026-06-13, статус: accepted, грилл)
**Контекст:** `F` измерялся в массе/деньгах/спросе вперемешку; два бюджета M и B (уточняет D-021, §4.2–4.3, §5.2, §10).
**Решение:** единственный ledger — **деньги**. `F` = денежная стоимость импорта за окно (нелокализованный спрос + хвосты обслуживания, по каталогу). Зажим — `F` (деньги) vs `M` (деньги). **Масса-бюджет `B` упразднён** как отдельная сущность; масса кусается только через цену (`масса×c`). Нет отдельного лимита пусковой массы.
**Альтернативы:** деньги + жёсткий лимит пусковой массы (отклонено — двойная бухгалтерия/UI; тезис «дороговизны» доходит через деньги).

## D-028 — MES — один закон, два гейта   (2026-06-13, статус: accepted, грилл)
**Контекст:** §6.3 говорил «у навыков тоже MES» — один концепт или два (уточняет D-015).
**Решение:** MES — один закон (минимальный эффективный масштаб *способности*), гейтит узел на **двух фронтах**: (1) `D ≥ process-MES` (масштаб установки) и (2) колония тянет конвейер обучения для его **специалистов** (у обучения свой MES). Навыки — просто ещё один вход узла, сам MES-гейтнутый. «Специалист» — каноничный термин.
**Альтернативы:** дефицит навыков как отдельная подсистема (отклонено — теряется единый закон «масштаб против размера колонии» на всех осях).

## D-029 — Спрос = потребление + производный   (2026-06-13, статус: accepted, грилл)
**Контекст:** §1.4 требует агрегации спроса вверх, прототип брал `D ∝ населению` (агрегации нет).
**Решение:** `D` узла = **потребительский** (население × на-душу, на конечные узлы) + **производный** (локализация узла создаёт спрос на каждый его вход). Апстрим-спрос = сумма производного от всех локализованных потребителей — это и есть агрегация §1.4 (делает capability-push §3.5 выгодным). Импорт готового товара даёт ноль производного спроса. Прототип это опускал.
**Альтернативы:** спрос только от населения (отклонено — §1.4 становится ложью, фирменной петли нет).

## D-030 — Население — рычаг игрока; рождения гейтнуты инфраструктурой   (2026-06-13, статус: accepted, грилл)
**Контекст:** §6.1/§10.2 vs авто-рост прототипа (резолвит §10.6/§4.7).
**Решение:** население меняется только (1) **завозом колонистов** (покупаются как груз: разовая цена посадки + **вечный шлейф спроса** в `F`) и (2) **рождениями — только при построенной и содержимой инфраструктуре** (роддома, школы — сами MES-узлы с хвостами и специалистами). Нет авто-иммиграции и бесплатных рождений. Рост — всегда бюджетный размен; иногда оптимально оставаться маленькими.
**Альтернативы:** авто-рост по излишку (отклонено — рост перестаёт быть решением).

## D-031 — Отмена субсидии = земная инфляция + земные события   (2026-06-13, статус: accepted, грилл)
**Контекст:** триггер концовки «отмена» был не определён (уточняет D-017/D-021, резолвит §10.6).
**Решение:** субсидия `M` номинально фиксирована, но **реально стачивается земной инфляцией** (экзогенный драйвер зажима §10.4 — триллион сжимается, даже если колония стоит) и **режется земными событиями** (кризис, смена политики) — обе причины вне контроля колонии. Полная отмена = земное событие/обесценивание до незначимости. Не приговор за бесплодность → самодостаточность остаётся дебриф-онли (D-025).
**Альтернативы:** отмена по бесплодности с ультиматумом (отклонено — выбрана макроэкономическая, более бессильная и тематичная); чистый таймер/только событие (отклонено — инфляция даёт постоянный экзогенный гнёт).

## D-032 — Импорт-эскейп универсален; не-покупаемых узлов нет   (2026-06-14, статус: accepted, грилл)
**Контекст:** §1.2 оставлял лазейку «узлы под эмбарго/уникальные купить нельзя», противореча несущему импорт-эскейпу (резолвит §3.6).
**Решение:** купить можно ВСЁ — нет принципиально не-покупаемых узлов. Невозможно бывает только локальное производство (чёрный узел); доступ за деньги универсален. Тезис — экономическая невозможность (вечный разорительный расход), а не блокада/отказ. «Эмбарго» = временный ценовой/объёмный шок (земное событие, D-031), не статус узла.
**Альтернативы:** настоящие не-покупаемые узлы (механика блокады) — отклонено: смещает посыл с дороговизны на политический отказ, ломает импорт-эскейп и раскрытие сверху-вниз.

## D-033 — Мегапроект: стройка игрока + оптимистичная проекция   (2026-06-14, статус: accepted, грилл)
**Контекст:** прототип строил термояд авто; §11.5 — виден ли потолок заранее (резолвит §11.5).
**Решение:** мегапроект — осознанная стройка игрока, финансируемая накоплением из излишков по окнам (дорого, конкурирует с локализацией). Эффект подаётся оптимистично смещённой проекцией — той же машинерией, что скрытый MES/оценки узлов (D-014, §3.4): игрок верит, что это прорыв; истинный потолок вскрывается только постройкой и эксплуатацией. Так мегапроект — осязаемая ложная надежда Акта 2; дебриф называет его отсрочкой.
**Альтернативы:** честно-известный потолок (тактическая заглушка) — отклонено: убивает ложную надежду, ключевой момент §11; авто-стройка (как в прототипе) — отклонено: рост/спасение должно быть решением игрока.

## D-034 — Специалисты = статистические пулы; bus factor виден   (2026-06-14, статус: accepted, грилл)
**Контекст:** поимённые личности vs пулы навыков (резолвит §6.7).
**Решение:** специалисты моделируются статистическими пулами навыков (не поимённо; имена — опциональный флавор) — трактуемо, в жанре системного сима, без скоуп-крипа управления персонажами. На критичных узлах **bus factor виден** (штат/покрытие показаны — это НЕ скрытый MES, уточнение к D-014); скрыт лишь риск/последствие тонкого покрытия. Единичная смерть теряет именную драму — компенсируется системным ужасом каскада (§5.6).
**Альтернативы:** поимённые личности (отклонено — отдельная система персонажей, уводит от системной трагедии); полностью скрытый bus factor (отклонено — штат игрок объективно знает; скрывать нечестно).

## D-035 — Раскрытие тумана: пассивное (бесплатно) + платная разведка   (2026-06-14, статус: accepted, грилл)
**Контекст:** чем платится разведка и какова гранулярность (резолвит §3.6).
**Решение:** два механизма. (1) Бесплатное пассивное раскрытие: коммит на локализацию узла вскрывает следующий тир его рецепта. (2) Платная разведка: деньги (единый ledger, D-027) + окно лага (§4.4), чтобы вскрыть рецепт узла и сузить оптимистичную вилку оценки (§3.4); по одному узлу за раз, глубже накопительно дорого. Разведка конкурирует за те же деньги, что F/капитал → под зажимом игрок недо-разведывает и влетает в скрытые чёрные узлы (ловушка, §3.3, §8.7).
**Альтернативы:** только пассивное раскрытие действием (отклонено — нельзя планировать, опаково/наказывающе); бесплатная полная разведка (отклонено — снимает размен прозорливость-vs-бюджет, убивает поздний сюрприз).

## D-036 — Нарратив: диегетический фрейминг без интерпретации   (2026-06-14, статус: accepted, грилл)
**Контекст:** чистые системы vs нарративная обёртка; рука дебрифа (резолвит §7.7).
**Решение:** единственный нарративный слой — диегетический: статистика + земные новости (через них всплывают инфляция/события D-031) + репорты колонии. Без авторских персонажей и арок (согласуется с пулами D-034). Игра **не интерпретирует и не морализирует**: констатирует факты (включая запас хода в дебрифе, D-025), вывод делает сам игрок — чистейшая форма «заслуженного прозрения» (§2.5).
**Альтернативы:** интерпретирующий дебриф с эпилогом/аннотациями «знаки были здесь» (отклонено — морализирует за игрока); чистые системы вовсе без диегетики (отклонено — слабее эмоция); персонажи с арками (отклонено — против системной трагедии, D-034).

## D-037 — Калибровка: стена индустриальная, не транспортная   (2026-06-14, статус: accepted, калибровка)

Собраны реальные опорные данные (`references.md` §3–4): доставка на Марс ~$400/кг (Starship-цель) … $5 800/кг (Falcon Heavy); фаб чипов $15–20 млрд и >17 млн пластин/год; завод API 200–500 т/год; КПД ECLSS вода 0,98 / O₂ >0,95. Заавторен граф из ~28 узлов (`documents/graph.md`), прокручен в прототипе.
**Решение:** при честной Starship-экономике доставка балка **дёшева** → тирания Циолковского (D-012) — *слабый* зажим. «Фантастическая дороговизна» при реальных числах исходит почти целиком из **внутренней стоимости + недостижимого MES чёрных узлов** (фаб/фарма/катализаторы), а не из массы-на-орбиту. Это **усиливает** тезис (стена индустриальная — её не убрать удешевлением ракет), но смещает акцент §4 с уравнения ракеты. Калиброванные ручки прототипа: `c`=5e3, `capital_factor`=5e7 («колено», где доступность капитала перестаёт связывать и берёт верх MES-гейт). Подтверждено численно: автономия плато 67–78% (<100%), **запас хода пригвождён к ~0.5 окна в каждом прогоне** при любом `c`/`M`/термояде — разрыв D-025 устойчив; F/M подползает к/за 1.0 (давление Акта 2); термояд держит F/M ниже, но автономию не двигает (отсрочка, не решение, §11).
**Альтернативы:** держать высокий `c` ради сильного Циолковского-зажима (отклонено — нечестно против Starship-данных; тезис не должен зависеть от дорогих ракет); считать модель опровергнутой дешёвой доставкой (отклонено — наоборот, MES-стена робастна к `c`, что и есть суть). **Открыто:** возможный пересмотр текста §4 (`mechanics.md`) под смещённый акцент. **Уточнено D-038.**

## D-038 — Пусковая мощность — класс капитала, а не цена `c`   (2026-06-14, статус: accepted, калибровка)
**Контекст:** уточнение D-037/D-012. Дешёвый Starship-$/кг (~$400) — амортизированная стационарная цифра; марсианский груз бёрстен (~5 дней раз в 26 мес). Магическая константа `c` скрывала реальную природу зажима.
**Решение:** упразднить `c` как заданную внешнюю цену доставки. Пусковая инфраструктура (башни, флот, криозаводы) — **класс капитала в едином ledger (D-027):** capex на постройку пропускной способности `K` (кг/окно) + **содержание/амортизация каждое окно, независимо от загрузки**. Переиспользование землянами ≈ 0 (флот под марсианский бёрст абсурдно велик для земного рынка) → знаменатель амортизации = только марсианский груз → простой 25/26 мес ложится целиком на программу. **Эффективный $/кг производный**, не заданный: `топливо·W + содержание·K`, делённое на завезённую массу `W`. Выпуклая цена (мой набросок выше) возникает эмёрджентно, навязывать формулу не нужно. **`K` — явный рычаг игрока** (D-018): сколько мощности строить. Переборщил → содержание давит каждое окно (идеальная ловушка: построил под ранний тяжёлый завоз, локализовал балк → мощность простаивает, но содержится). Недобрал → не продавишь завоз в кризис. Потолок чисто экономический, масса-бюджет `B` не воскресает (D-027 цел) — прямой параллелизм с D-032 («прокачать можно любой тоннаж, стена — капитал простоя, не блокада»). Так D-012 получает реальные зубы: тирания = капитал бёрст-мощности без переиспользования, а не уравнение ракеты.
**Альтернативы:** выпуклая цена `c_eff(W)` навязанной формулой (отклонено — D-038 даёт то же эмёрджентно и честнее, без магической константы); авто-амортизация без рычага игрока (отклонено — теряется стратегическое решение «сколько строить», менее тезисно, D-018); жёсткий лимит тоннажа окна (отклонено — воскрешает масса-бюджет `B`, нарушает D-027).

## D-039 — Технология запуска — лестница игрока (транспортный «ложный спаситель»)   (2026-06-14, статус: accepted)
**Контекст:** §4 объяснял зажим через уравнение ракеты и подачу формулы игроку; D-038 сделал доставку капиталом. Игрок должен *прожить* транспортный урок, а не получить его лекцией. Заодно §4 — место инокулировать против самого частого контртезиса «Starship всё удешевит» (Зубрин/Маск, references §6).
**Решение:** пусковая способность — короткая **лестница технологий, по которой игрок идёт сам**, поверх капитальной модели D-038:
- **Поколение 0 — одноразовые (брют-форс Циолковского):** стэк выбрасывается каждый пуск → стоимость растёт ~линейно с тоннажем, нет амортизации, нет переиспользования. Построить «сотню ракет» — осязаемый **экономический тупик**, который игрок открывает действием (соблазн → сомнение). Это диегетическая подача уравнения ракеты — **показ, а не объяснение** (резолвит §4.7 «насколько явно показывать уравнение»).
- **Поколение 1 — многоразовость + орбитальная дозаправка + ISRU-топливо обратного пути (Starship-класс):** крупный R&D-capex (транспортный мини-мегапроект, подаётся оптимистично, как D-033), затем предельный $/кг обрушивается к топливному полу. **НО:** (а) стена бёрст-мощности D-038 остаётся (флот всё равно нужен под бёрст окна, простаивает 25/26 мес); (б) индустриальной стене MES (D-037) это не помогает ничем. «Удешевляет, но не решает» — в обоих смыслах.
- **Синтез (§4.6):** даже игрок, полностью выигравший транспорт (дешёвый многоразовый Starship + ISRU), упирается в индустриальную стену — запас хода остаётся пригвождён к ~0.5 (D-025). **«Выиграть ракетную войну, проиграть индустриальную»** — experiential-опровержение Starship-оптимизма.
- **Сцепка с графом:** локализация узла `fuel`/ISRU (`graph.md`) — внутриграфовый крючок, открывающий/удешевляющий Поколение 1 (петля §4.5 назад в граф).
**Альтернативы:** единственная абстрактная ручка пусковой мощности (D-038 в одиночку) — отклонено, но сохранена как субстрат, обогащённый лестницей; развесистое дерево технологий — отклонено (скоуп-крип, двух поколений достаточно); подача уравнения ракеты тултипом — отклонено (D-010 «заслуженное прозрение», показ а не рассказ).

## D-040 — Стек и архитектура: web/TS, движок ⟂ UI, вертикальный срез   (2026-06-14, статус: accepted)
**Контекст:** дизайн прошёл стадию «доказать тезис на бумаге+песочнице» (D-037…D-039); пора проектировать реальную кодовую базу. Прототип — throwaway (Python).
**Решение:** платформа — **веб, TypeScript** (игра-аргумент должна легко шариться). Несущий принцип — **разделение движка и UI**: движок (`src/engine/`) — чистый TS без UI-зависимостей, детерминирован (сид-RNG), полностью юнит-тестируем, портирует economy.py; это и есть аргумент → корректность критична (золотые тесты на числа прототипа: автономия 67–77%, запас хода 0.5, эфф. $/кг ~$1M). UI (`src/ui/`) — тонкий реактивный слой, фреймворк **изолирован и заменяем** (низкий риск). Сборка Vite+Vitest, сейв в localStorage. Старт — **вертикальный срез** (одна петля наскозь: дашборд→drill-down→манифест→ход→дебриф), а не вся система сразу. План — `implementation.md`.
UI — **Lit** (веб-компоненты, ~5КБ, стандартные Custom Elements, реактивный рендер): легче полноценного фреймворка, но структурнее vanilla (панели §8 — самостоятельные компоненты); игра пошаговая, перерисовка редкая, состояние централизовано в движке. Решение изолировано разделением движок ⟂ UI.
**Альтернативы:** desktop (Tauri/Electron) — отклонено для старта (хуже шарится, тезис должен доходить до людей); Python+TUI — отклонено (слабее как широко-шаримая игра); полная архитектура до кода — отклонено (дольше до играбельности; срез режет риск); React/Svelte — отклонено (тяжелее Lit для пошаговой игры с редкой перерисовкой); чистый vanilla — отклонено (Lit даёт структуру компонентов почти бесплатно).

## D-041 — Микро-интеракция: поштучный заказ слайдерами, табы Земля/Марс   (2026-06-14, статус: accepted)
**Контекст:** макро-петля (Фазы 0–6: «локализуй узел + колонисты») честна, но не тактильна — игрок не чувствует колонию.
**Решение:** player-facing модель — **поштучные заказы** (тонны/штуки) слайдерами на карточках с иконками, разнесённые по **табам**: планирование на Земле (заказ завоза) + планирование на Марсе (очередь стройки). Живые подсказки-балансы и лимит массы. Детали — `colony-sim.md`.
**Альтернативы:** оставить макро-манифест (отклонено — не тактильно); только UI-обёртка без новых балансов (отклонено — «макро»-ощущение осталось бы).

## D-042 — Полный физический слой: стоки/потоки ресурсов + каскад дефицита   (2026-06-14, статус: accepted)
**Контекст:** интерактивность требует ощутимых физических последствий, которых нет в денежно-абстрактном движке.
**Решение:** над денежным ledger (D-027 цел) — **по-ресурсный учёт стоков/потоков**: энергия (баланс ген↔спрос), еда, вода, газы, материалы/спейрс; `сток' = сток + производство + прибытие − потребление`; дефицит каскадит (браунаут→срез выпуска→голод/смертность/поломки). КПД рециклинга η<1 (ECLSS). Запас хода (D-025) теперь считается из реальных стоков. `colony-sim.md` §2.
**Альтернативы:** лёгкие балансы только энергии/еды (отклонено пользователем в пользу полного сима); чистые деньги (отклонено — не ощутимо).

## D-043 — Явная логистика запуска: площадки/ракеты/топливо   (2026-06-14, статус: accepted, уточняет D-038/D-039)
**Контекст:** абстрактная капитальная ручка `K` (D-038) не тактильна.
**Решение:** пусковая мощность — **явные площадки** (капитал, простаивают 25/26 мес — бёрст D-038), каждая 1 пуск/день → 5 пусков/окно (старт 5 площадок = 25 пусков); **ракеты** (payload/пуск; классические одноразовые Gen0 → многоразовые+орбитальная заправка Gen1 после RnD, D-039); **топливо** (расходник). Пропускная способность/окно = площадки×пуски×payload — заказ нельзя превысить (строй площадки или режь завоз). `colony-sim.md` §5.
**Альтернативы:** оставить абстрактный `K` (отклонено — не тактильно); жёсткий масса-бюджет `B` (отклонено — D-027, потолок остаётся экономическим через стоимость площадок/простоя).

## D-044 — Стройка на Марсе: структуры с пререквизитами и энергобалансом   (2026-06-14, статус: accepted)
**Контекст:** «локализуй узел» абстрактно; нужны осязаемые здания и зависимости.
**Решение:** на Марсе строятся **структуры** (узлы с capex/апкипом/энергобалансом/штатом/пререквизитами): электростанции (ядерные → площадка отходов), фермы (энергия+вода+семена), ЖО-модули (рециклинг η<1), медблоки (тянут фарма-чёрный), роддома (→ рождения D-030), RnD-лаборатории (лестница D-039/мегапроекты D-033), школы (обучение специалистов D-028), заводы локализации (узлы графа; чёрные — нельзя), посадочные площадки, хранилища отходов. `colony-sim.md` §6.
**Альтернативы:** только заводы-узлы без опорных структур (отклонено — теряются энергия/ЖО/отходы-зависимости, несущие физический слой и тезис).
